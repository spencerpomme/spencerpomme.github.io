create_maxheap的时间复杂度分析

粗略分析：loose bound

首先来回顾一下代码：

第一眼看起来，这个函数的时间复杂度应该是O(n * O(max_heapify2))，而max_heapify的时间复杂度在之前的blog中已经分析过了，是O(lg n)。所以至少可以得到一个大致的时间复杂度应该是o(n*lg n)。（O表示tight upper bound，而o则不是）。然而我们可以做得更加精确一些。

我们已经知道，对于一个有n个节点的heap来说，其中某个节点调用max_heapify的时间复杂度为O(h)，h为该节点的height。因此，每一层调用max_heapify执行的工作量是不一样的。譬如在height=1的那一层（即leaf节点层的上一层）调用max_heapify的步骤为1c，而在root的调用时间则是lg(n)c，c是常数。

所以为了知道总的时间，还需要知道每一层的节点数与height的关系。

每一层的节点数与height的关系

注意: 推导时，假设heap构成了一棵full binary tree。

解释: 这是为了求得每一层最多时候的情况而规定的，同时当n趋近于\(\infty\)的时候，leaf层是否满足full binary tree的定义对得到的规律而言无影响，但是会大大简化推导的过程。

对于一棵节点总数为n的full binary tree来说，其每一层的节点数是一个\(r=2\)的等比数列，所以节点数之和与深度depth的关系为： $$ \begin{eqnarray*} \large n &=&\sum_{d=1}^{\lg n} r^d = \frac{1-r^d}{1-r} \ \ \ \ \ (r\neq1)\\\\ &&带入r=2, 有:\\\\ &=& \frac{1-2^d}{1-2}\\ &=& 2^d-1 \end{eqnarray*} $$ 其中，d为该层的节点的深度(depth)。对于一棵full binary tree我们知道，节点的depth和height存在关系：\(d+h=\lg n\)，其中n为总的节点数，h为height。所以(A)式可以改写为： $$ \large n \normalsize = 2^{\lg n - h} - 1\ \ \ \ \ $$ 现在来求\(h = h_{0}\)时，该层有多少个节点。方法很简单，只需要用从顶点到\(h_{0}\)层的总节点数减去从顶点到\(h_{0} +1\)层的总节点数即可算出： $$ \begin{eqnarray*} S_{0} &=&(2^{\lg n - h_{0}} - 1) - (2^{\lg n - (h_{0}+1)} -1)\\ &=&\frac{2^{\lg n}}{2^{h_{0}+1}}\\ &=&\frac{n}{2^{h_{0}+1}}\\ \end{eqnarray*} $$ 由此，可以证明每一层最多含有\(\lceil\frac{n}{2^{h+1}}\rceil\)个节点。（这个证明是CLRS中的一道习题）

推导create_maxheap的时间复杂度

证明了对于一个heap来说每一层最多有\(\lceil\frac{n}{2^{h+1}}\rceil\)个节点之后，就可以进一步推导create_maxheap的时间复杂度了。有： \[ T(h) = \frac{n}{2^{1+1}}(1c) + \frac{n}{2^{2+1}}(2c) + \cdots + \frac{n}{2^{\lg(n)}}(\lg(\frac{n}{2})\cdot c) + \frac{n}{2^{\lg(n)+1}}(\lg(n)\cdot c) \ \ \ \ \ (a) \] 其中，\(h\in[1,\lg(n)]\)闭区间。这个很容易理解，当h也就是height为1的时候，就是leaf层的上一层；当h为\(\lg(n)\)的时候，就是root节点了。(a)式的每一项由两个部分组成，第一部分为层节点数，第二部分为每个该层节点需要做的work，c为一个常数。

对于(a)式提取公因式\(\frac{cn}{2^{1+1}}\)，可以得到： \[ T(h) = \frac{cn}{4}\cdot (\normalsize 1 + \frac{2}{2} + \frac{3}{4} + \frac{4}{8} + \frac{5}{16} + \cdots + \frac{h}{2^{h-1}}) \ \ \ \ \ (b) \] 即： \[ T(h) = \frac{cn}{4}\cdot (1 + \sum_{h=1}^{\lg(n)} \frac{h}{2^{h-1}}) \ \ \ \ \ (c) \] 容易发现，\(\sum_{h=1}^{\lg(n)} \frac{h}{2^{h-1}}\)这一项可以拆解为等差数列和等比数列的商，进一步简化可以写为\(\sum_{h=0}^{\lg(n)} \frac{h}{2^{h}}\)，然后当： \[ \sum_{h=0}^{\infty} \frac{h}{2^{h}} \ \ \ \ \ (d) \] (d)式满足公式：\(\sum_{k=0}^{\infty} kx^k = \frac{x}{(1-x)^2}\)（公式证明可见维基百科），当且仅当\(\vert k \vert < 1\)时成立。这里\(k = \frac{1}{2}\)，因此(d)式收敛于： \[ \sum_{h=0}^{\infty} \frac{h}{2^h} = \frac{1/2}{(1- 1/2)^2} = 2 \ \ \ \ \ (e) \]

于是结合(e), (c)两式有： \[ \begin {eqnarray*} \large O(\normalsize \frac{cn}{4}\cdot(1+\sum_{h=1}^{\lg(n)} \frac{h}{2^{h-1}})\large)\normalsize &=&\large O(\normalsize\frac{cn}{4}\cdot(1+\sum_{h=0}^{\infty} \frac{h}{2^{h}})\large) \\ &=&\large O(\normalsize\frac{cn}{4}\cdot(1+2)\large)\\ \\ &=&\large O(\normalsize\frac{3cn}{4}\large)\\ \\ &=&\large O(\normalsize n \large) \end {eqnarray*} \] 至此，证明完毕。可知create_maxheap的时间复杂度是线性的。

heapsort算法

同样，这个部分也分为两个子内容，首先实现heapsort算法，然后分析其时间复杂度。

算法实现

在之前的max_heapify函数和create_maxheap函数的基础上，实现heapsort算法就比较简单了。用自然语言描述的话如下：

  1. 对一个无序array调用create_maxheap函数，将其转化为一个max heap
  2. 找到array中的最大元素，即array[1] (根据定义max heap的root上的元素是整个array中最大的)
  3. 交换array[1]和最后一个元素，这样最大的元素到了array的最后
  4. 通过将array的长度减小1来把最后一个元素丢出array
  5. 对缩小长度后的array从第一个元素（亦即heap root）调用max_heapify函数
  6. 判断array的长度是否>1，若是则从步骤2继续，否则结束循环

经过这个过程之后，原无序array中的元素就会以非递减的顺序排列。其本质上就是将在root位置的当前最大元素和当前最后一个元素交换位置，然后再重构整个heap使得缩小长度by 1的那个array再次满足max heap的定义。这样就实现了排序的目的。代码如下：

heapsort算法的时间复杂度分析

分析heapsort的算法时间复杂度，需要看起算法每一步的时间复杂度。

  1. \(O(n)\)
  2. 一次循环需要\(O(1)\)， 总共要循环 \(n-1\) 次，因此是\(O(n)\)
  3. 一次循环需要\(O(1)\)， 总共要循环 \(n-1\) 次，因此是\(O(n)\)
  4. 一次循环需要\(O(1)\)， 总共要循环 \(n-1\) 次，因此是\(O(n)\)
  5. 一次循环需要\(O(\lg n)\)， 总共要循环 \(n-1\) 次，因此是\(O(n\cdot \lg n)\)

所以heapsort算法的时间复杂度为\(O(n\cdot \lg n)\)，且易知空间复杂度为\(O(n)\)，即in place。
